# BGOBS 0.3.19

BGOBS is now presented and maintained as a public, open-source OBS Studio
plugin. Its background-removal and enhancement filters work with ordinary OBS
video sources; the separate, privately developed CaCam Android app remains an
optional source integration.

## Highlights

- Make English the default language for public project documentation while
  retaining OBS interface translations.
- Add contributor, security, architecture, release, attribution, and
  third-party documentation.
- Publish a standalone version 1 CaCam USB protocol specification that does not
  require access to CaCam's private implementation.
- Harden direct USB parsing, shutdown, and reconnection against malformed input
  and stale worker callbacks.
- Bound ADB HTTP and update-metadata requests with response limits and timeouts;
  require HTTPS for release metadata and keep update checks off OBS-sensitive
  callbacks.
- Reuse video textures, improve CPU fallback behavior, and remove data races in
  shared source state.
- Expand Rust, native, sanitizer, and documentation-site checks.

## Compatibility and downloads

The direct USB wire format remains protocol version 1. Existing 0.3.x scene and
preset settings are intended to remain compatible.

A source tag alone does not certify a binary on every operating system. Use
only the packages actually attached to the GitHub release after their platform
build and smoke test succeed. Packages absent from the release page are not
claimed as verified for 0.3.19.

The release workflow targets:

- Windows x64, as an explicitly unsigned runtime ZIP plus debug symbols;
- macOS 12 or later on Intel and Apple silicon, as a universal package plus
  debug symbols. When Apple signing credentials are unavailable, the filename
  ends in `-unsigned.pkg` and Gatekeeper may require an explicit **Open Anyway**
  decision after checksum verification;
- Ubuntu 24.04 x86_64, as an explicitly unsigned Debian package plus a debug
  symbols package.

The release also includes a SHA-256 manifest, an SPDX SBOM, and GitHub artifact
attestations. Verify the exact assets shown on the release page rather than
predicting a download URL.
